# 프론트엔드 변경 이력 (Frontend Modification History)

## 2026-03-25 17:55:00

- **작업 내용:** 프론트엔드 통신 안정성 및 회복력(Resilience) 강화
- **상세 변경 내역:**
  - **`apiClient.ts` 고도화:**
    - **타임아웃 도입:** `AbortController`를 활용하여 모든 API 요청에 30초 타임아웃을 적용하고, 초과 시 사용자에게 명확한 에러 메시지를 전달하도록 개선했습니다.
    - **자동 재시도(Retry) 로직:** 네트워크 장애나 5xx 서버 에러 발생 시 최대 3회까지 지수 백오프(Exponential Backoff) 방식으로 자동 재시도를 수행합니다. (지연 시간: 1초 -> 2초 -> 4초)
    - **토큰 리프레시 뮤텍스(Mutex):** 여러 요청이 동시에 401 에러를 발생시킬 때, 단 한 번만 토큰 갱신 요청을 보내고 나머지 요청은 그 결과를 기다려 재사용하도록 뮤텍스 패턴을 적용했습니다.
  - **SSE 연결 안정화 (`NotificationSubscriber.tsx`):** 알림 수신용 SSE 연결이 끊겼을 때, 최대 10회까지 지수 백오프 방식(최대 30초 지연)으로 재연결을 시도하여 실시간 알림의 연속성을 확보했습니다.
  - **환경 변수 정비 (`frontend/.env`):** 외부 API 연동을 위한 `VITE_API_URL` 환경 변수 설정을 추가 및 정비했습니다.
- **결과/영향:** 네트워크 환경이 불안정한 상황에서도 앱이 스스로 복구될 수 있는 능력이 강화되었으며, 불필요한 인증 요청 중복을 방지하여 서버 부하를 줄였습니다.
