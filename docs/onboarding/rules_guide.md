# 🛠️ DayPoo 팀원 협업 규칙 가이드

이 문서는 DayPoo 프로젝트에서 팀원들이 원활하게 코드를 병합하고 리뷰하기 위해 **자동으로 강제되는 핵심 규칙(Lint & Commit)**을 설명합니다.

여러분은 코드를 짜기만 하면 됩니다! 나머지는 봇(Husky)이 알아서 검사해 줍니다. 🤖

---

## 1. 커밋 메시지 규칙 (Commitlint)

커밋 메시지는 반드시 `타입: 제목` 형식으로 작성해야 합니다. 이 양식을 지키지 않으면 **커밋 자체가 실패(Reject)**합니다.

### ✅ 올바른 예시

- `feat: 유저 로그인 API 추가`
- `fix: 카카오맵 마커 안 뜨는 버그 수정`
- `style: 메인 페이지 버튼 색상 변경`
- `docs: README.md 시스템 아키텍처 다이어그램 업데이트`

### ❌ 틀린 예시 (커밋 실패)

- `로그인 추가` (타입이 없음)
- `Feat: 로그인 추가` (타입은 소문자여야 함)
- `feat : 로그인 추가` (콜론(`:`) 뒤에만 띄어쓰기해야 함)

### 📌 사용 가능한 타입 목록

| 타입       | 용도                                                              |
| ---------- | ----------------------------------------------------------------- |
| `feat`     | 새로운 백엔드 API, 프론트엔드 UI 컴포넌트 등 **새로운 기능** 추가 |
| `fix`      | 잘 못 동작하던 **버그나 오류** 수정                               |
| `docs`     | README, Swagger 주석 등 **문서만** 변경                           |
| `style`    | CSS, 디자인 등 UI 적인 포맷 변경 (로직 수정 없음)                 |
| `refactor` | 코드 리팩토링 (기능은 그대로인데 내부 구조만 개선)                |
| `test`     | 테스트 코드 추가 또는 수정                                        |
| `chore`    | 패키지 버전 업데이트, 의존성 설치, 빌드 세팅 수정                 |

## 2. 코드 포맷 자동 정리 규칙 (Lint-staged)

개발하시고 `git commit`을 누르는 순간, 여러분이 **방금 수정한 폴더의 언어에 맞춰서 코드 스타일 검사관이 자동으로 작동**합니다. 만약 치명적인 문법 오류(예: 선언되지 않은 변수 사용)가 발견되면 커밋이 막히니 에러를 수정해 주세요!

### ⚛️ 프론트엔드 (`/frontend` - React/JS)

**적용 도구**: ESLint + Prettier

- **문법 검사**: 쓰지 않는 변수 금지, `console.log` 지양.
- **코드 줄 맞춤**:
  - 탭(Tab) 대신 **스페이스 2칸** 사용.
  - 따옴표는 무조건 **싱글 쿼터(`'`)** 사용. (`"hi"` ➔ `'hi'`)
  - 문장 끝에는 무조건 **세미콜론(`;`)** 붙임.
  - 한 줄 코드가 **100자**를 넘어가면 알아서 예쁘게 다음 줄로 쪼개줍니다.

### 🍃 백엔드 (`/backend` - Java/Spring)

**적용 도구**: Google Java Format (Spotless Gradle 플러그인)

- **문법 검사**: 사용하지 않는 `import` 구문 전부 자동으로 삭제!
- **코드 줄 맞춤**: 구글(Google) 사내 자바 표준에 맞추어 들여쓰기와 중괄호(`{}`) 위치를 강제 정렬합니다.

---

## 3. 브랜치 생성 규칙

- 브랜치명은 **원하는 이름으로 자유롭게** 생성하고 작업할 수 있습니다.
- 단, 혼동을 막기 위해 기능 개발은 `feature/...`, 버그 수정은 `hotfix/...` 로 시작하는 것을 "권장"합니다. (강제 아님!)

> 💡 **Tip:** "앗, 내가 코드를 좀 더럽게 짰나?" 걱정하지 마세요. 에러만 안 내면, 커밋할 때마다 Prettier, Spotless가 **여러분의 코드를 업계 최고 표준 모양으로 알아서 착착 예쁘게 다림질해 줍니다!** 편하게 코딩하세요!
