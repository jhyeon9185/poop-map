# 리뷰 전체 보기 오류 수정 및 작성 권한 강화 계획

## 🎯 목표
1. '전체 후기 보기' 클릭 시 발생하는 런타임 오류(화이트아웃 또는 에러 페이지)를 해결합니다.
2. '후기 작성' 권한을 방문 인증을 완료한 사용자로 제한합니다.
3. 백엔드에서 필요한 수정을 사용자에게 안내합니다.

## 🛠 작업 단계

### Phase 1: `ReviewListModal.tsx` 런타임 오류 수정
- [ ] `summaryData.avgRating`이 `null`일 경우 `toFixed(1)` 호출 시 발생하는 오류를 방지하기 위해 널 체크 및 기본값(`0.0`) 처리를 추가합니다.
- [ ] 기타 데이터가 로드되지 않은 상태에서 접근할 가능성이 있는 속성들에 옵셔널 체이닝(`?.`)을 적용합니다.

### Phase 2: `ToiletPopup.tsx` 후기 작성 권한 제한
- [ ] `handleOpenReviewModal`에 방문 여부(`toilet.isVisited`) 체크 로직을 추가합니다.
- [ ] 방문하지 않은 사용자에게는 '방문 인증 후에만 리뷰를 남길 수 있습니다.'라는 안내를 제공합니다.

### Phase 3: 백엔드 지원 권장 사항 정리
- [ ] 현재 백엔드에서 화장실 목록 조회 시 각 화장실의 방문 여부를 반환하지 않고 있으므로, 이를 연동하기 위한 백엔드 API 고도화가 필요함을 사용자에게 설명합니다. (규칙에 따라 백엔드 코드는 직접 수정하지 않음)

### Phase 4: 최종 점검 및 기록
- [ ] 수정 사항이 정상적으로 반영되었는지 확인하고 `docs/frontend-modification-history.md`에 기록합니다.
