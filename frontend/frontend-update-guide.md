# 랭킹 페이지 프론트엔드 수정 가이드 (UI/UX 개선 제안)

랭킹 시스템의 백엔드 로직이 '보유 포인트'에서 '활동 지표 기반 복합 점수'로 전면 리팩토링됨에 따라, 프론트엔드 UI에서도 아래 사항들을 업데이트할 것을 권장합니다.

## 1. 데이터 레이블 및 설명 업데이트
현재 프론트엔드에는 랭킹 기준이 '인증 횟수'로 하드코딩되어 있습니다. 이를 새로운 점수 체계에 맞게 수정해야 합니다.

- **대상 파일**: `frontend/src/pages/RankingPage.tsx`
- **수정 사항**:
  - `TAB_CONFIG`의 `desc` 필드:
    - `total`: "누적 인증 횟수 기준" → "기록 인증 + 장소 방문 점수 합산"
    - `local`: "현재 위치 기반 지역 랭킹" → "현재 위치 기반 지역 활동 점수"
  - `scoreLabel` 매핑:
    - 현재 `tab === 'health' ? '점' : '인증'`으로 분기되어 있으나, 전체/지역 랭킹도 이제 점수 개념이므로 모두 **"점"**으로 통일하는 것을 권장합니다.

## 2. 아이템 팝업 및 상세 정보 연동
현재 유저 상세 팝업(`ItemPopup`)에 표시되는 아이템과 칭호 정보 중 일부가 기본값으로 고정되어 있습니다.

- **아이템 목록**: `items: []`가 하드코딩되어 있습니다. 백엔드 응답의 유저 프로필 아이템(아바타, 마커 등)을 연동해야 합니다.
- **칭호**: `r.titleName`이 없을 때 "새내기 쾌변러"로 표시되는 기본값을 서비스 컨셉에 맞게 유지하거나, 백엔드 최신 칭호 시스템과 정합성을 맞춥니다.

## 3. 순위 변동 데이터 처리
- 현재 `change: 0`으로 하드코딩되어 있습니다. 백엔드에서 실시간 순위 변동 데이터를 제공하기 전까지는 UI에서 변동 아이콘을 숨기거나, '유지(-)' 상태로 고정 노출하는 처리가 필요합니다.

## 4. 백엔드 데이터 동기화 주의사항
백엔드 로직 변경 후, Redis 캐시에 이전 방식(포인트 기반)의 데이터가 남아있을 수 있습니다. 개발 환경에서는 백엔드의 `RankingService.initializeRankingsFromDb()`를 호출하여 데이터를 초기화한 후 테스트하시기 바랍니다.
