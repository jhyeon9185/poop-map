# Modification Logging Rule

## [CRITICAL] Modification Logging Protocol

모든 주요 수정 작업(기능 추가, 대규모 리팩토링, 아키텍처 변경 등)이 완료될 때마다 다음 규칙에 따라 로그를 남겨야 합니다.

1.  **대상**: 단순 오타 수정이나 사소한 주석 변경을 제외한 모든 유의미한 코드 및 기획서 변경 작업.
2.  **위치**: `docs/modification-history.md` 파일에 기록.
3.  **형식**: 역순(최신순)으로 기록하며, 다음 항목을 포함해야 함:
    - **날짜 및 시간**: YYYY-MM-DD HH:mm:ss 형식.
    - **작업 내용**: 수정된 주요 사항 요약 (기능 명칭 등).
    - **상세 변경 내역**: 어떤 파일이 어떻게 수정되었는지, 핵심 로직 변경점.
    - **결과/영향**: 작업 후 기대 효과 또는 주의사항.
4.  **수행 시점**: `notify_user`를 통해 사용자에게 작업 완료를 알리기 직전에 반드시 로그 작성을 완료해야 함.
